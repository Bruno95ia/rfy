# Exemplo Dashboard RFY (wireframe textual)

## Wireframe textual

[Header]
- Titulo do dashboard
- Acoes principais (relatorio, central de relatorios)

[Control Layer]
- Atualizacao + versao + filtro de periodo
- Atalhos operacionais (uploads, relatorio, configuracoes)

[Executive Canvas]
- Bloco A (Prioridade 1)
  - RFY Index (grande)
  - Receita Confiavel
  - Receita Inflada
  - Receita declarada

- Bloco B (Prioridade 2)
  - Proxima decisao (cliente, valor, acao)
  - Benchmark resumido

- Bloco C (Prioridade 2)
  - SUPHO (ITSMO, nivel, foco recomendado)

- Bloco D (Prioridade 2)
  - Alertas abertos (lista curta + expandir)

[Navegacao rapida sticky]
- Posicionamento
- Intervencoes
- Receita declarada vs confiavel
- Oportunidades em risco
- SUPHO
- Analises avancadas

[Analise operacional]
- Intervencoes prioritarias
- Forecast comparison
- Deal intelligence (table)

[Analise aprofundada]
- Painel executivo detalhado
- Unit economics / ICP
- IA status
- Analise detalhada (colapsada)

## Hierarquia recomendada
1. RFY + Confiavel + Inflada
2. Proxima decisao + Alertas + SUPHO
3. Comparativos e tabelas
4. Diagnostico profundo

## Regras de hierarquia
- Nenhum bloco abaixo pode repetir texto do topo sem nova informacao.
- SUPHO aparece no topo como diagnostico complementar, nao KPI principal.
- Dados historicos e tecnicos ficam no nivel de aprofundamento.
